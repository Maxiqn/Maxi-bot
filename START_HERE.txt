🚀 QUICK START (3 STEPS)
═════════════════════════════════════════════════════════════════

1️⃣  GET YOUR BOT TOKEN
   → Visit: https://discord.com/developers/applications
   → Create new application → Add bot → Copy token

2️⃣  SETUP PROJECT
   → npm install
   → cp .env.example .env
   → Edit .env with your token and Client ID

3️⃣  RUN BOT
   → node deploy-commands.js
   → npm start

🔧 CUSTOMIZATION
═════════════════════════════════════════════════════════════════

Create a new command:
1. Copy COMMAND_TEMPLATE.js → src/commands/mycommand.js
2. Edit with your logic
3. Run: node deploy-commands.js

Add database:
- MongoDB, PostgreSQL, MySQL, or Firebase
- Edit src/index.js to connect

Add more features:
- Message buttons
- Select menus
- Modal forms
- Voice channel integration


📖 FILE DESCRIPTIONS
═════════════════════════════════════════════════════════════════

Core Files:
  src/index.js          - Main bot file, loads commands, handles events
  src/config.js         - Bot configuration options
  src/constants.js      - Emojis, colors, messages
  src/utils.js          - Helper functions (formatting, validation)

Commands:
  src/commands/          - All slash commands go here
                          Each file is one command

Scripts:
  deploy-commands.js    - Registers commands with Discord
  setup.sh              - Automated setup script

Config:
  package.json          - Dependencies and scripts
  .env.example          - Template for environment variables
  .npmrc                - npm configuration
  .gitignore            - Git ignore rules


⚡ DEVELOPMENT VS PRODUCTION
═════════════════════════════════════════════════════════════════

Development (with auto-reload):
  npm run dev

Production:
  npm start

Both will run src/index.js


🆘 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════

See SETUP.md for detailed troubleshooting guide.

Quick fixes:
  • Bot not online → Check DISCORD_TOKEN in .env
  • Commands not working → Run: node deploy-commands.js
  • Module errors → npm install
  • Version mismatch → npm install  node@24


💡 TIPS & BEST PRACTICES
═════════════════════════════════════════════════════════════════

✓ Always use .env for sensitive data
✓ Never commit .env file to git
✓ Test commands in private server first
✓ Use npm run dev during development
✓ Run deploy-commands.js after modifying commands
✓ Check console for error messages
✓ Use /help command to show users what's available
✓ Add comments to your code
